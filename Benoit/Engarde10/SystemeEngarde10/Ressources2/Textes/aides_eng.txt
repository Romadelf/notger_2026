##aide_engarde_gene

                      Overview of ENGARDE version 10

Engarde 10 is a computer program for managing fencing competitions. 
It runs on Windows XP, Vista, 7, 8 and 10.

The program has two modes:
- a MINI mode which is free,
- a BASIC mode which is is a paid mode with more features 
- the PRO mode, is a paid mode with more features which implements all the specifications for the 
FIE competitions.

In order to run Engarde in the BASIC or PRO mode, a licence must be purchased, see the website http://engarde-escrime.com

##aide_diapoengarde_gene

                      Overview of DIAPOENGARDE version 9

DiapoEngarde is a computer program which displays on plasma screens, or other screens, 
results of a competition managed with Engarde. It runs on Windows 98 and later versions 
of Windows including XP and Vista.

DiapoEngarde produces an image on the computer screen which must be transferred to the
plasma screen (or other sort of screen) concerned. It is possible to transfer the image
to several screens. In that case, all these screens display the same image.

The program has two modes:
- an evaluation (Eval) mode which is free and allows to evaluate the program; in this mode, 
the size of the display is reduced and the duration of the diaporamas is limited;
- a PRO mode, a paid mode which has no limitation.

To run DiapoEngarde in the PRO mode, one needs to have an EngardePro running on the network
and having available "competition" licences.

"Competition" licences are purchased on the website http://engarde-escrime.com
"Competition" licences have a 3 days functioning: one day for preparation and two days
for running the competition.

##aide_diapoengarde_comp


                      DiapoEngarde functioning

DiapoEngarde has:
- the same organisation of windows than Engarde,
- the same display menus than Engarde (with some additional items),
- a diaporama mode which allows to have pages turning automatically.

DiapoEngarde opens competitions like Engarde: one has to browse the network to open the ".egw"
file of the chosen competition. One can also open a ".egw" file of a permanent copy when such
copy exists.
The competition or the copy to be opened must be located in a shared part of the computer.
When a good level of security is required, it is good to have the competition in an unshared 
area and a copy in a shared area.

In the toolbar of the documents, the button with a screen image allows launching the diaporama.
The esc key quits the diaporama mode.
In diaporama mode, a double click shows the toolbar during 10 seconds to do some tuning.
The toolbar contains buttons allowing tuning the display time of each page.

When modifications are made on fencers (new fencers, modifications of name...), it is necessary to close and reopen the competition so that DiapoEngarde takes them into account.

##aide_engarde_langues

                      LANGUAGES

Engarde is translated in several languages.

As the new texts of new versions are not immediately translated, it is possible to run Engarde
with two languages: a main and a secondary language.
The secondary language is used for messages not yet translated in the main language. It is
recommended to use French or English as secondary language. The choice "File | Langue - Language"
allows choosing the languages.

The help also works with a main and a secondary language. The choice "Help | Language of the help"
allows choosing the languages.

##aide_engarde_licences

                      LICENCES

An "EngardePRO" licence allows working in PRO mode with one session. When several simultaneous
sessions are envisaged, either on the same computer or on several computers of the network, 
it is necessary to purchase several licences.

The "EngardePRO" licences are embodied in keys which are automatically installed on the computer,
after been purchased, using the menu "Keys | Install a key". As key contains the MAC numbers of
the computer used for the action "Install the key"; it cannot be used on another computer.

When a key contains several licences, one can have several sessions of EngardePRO on the
computer of the key. The first session of Engarde sets directly Engarde in PRO mode. The
second set the BASIC mode. One has to do "Mode | Set PRO mode" to ask for a licence to the
first session. This can also be done from another computer of the network which has no key.
The first session is written Engarde(PRO main) in the top of the window. This session distributes 
licences. The menu "Network" allows to see the allocated sessions and the number of available 
sessions.

Engarde is completed by DiapoEngarde, a paid program which allows to display poules,
tableaux, rankings, etc. on a plasma TV screen.
DiapoEngarde needs a "competition" licence to run and needs an Engarde(PRO MAIN) session
to get this licence on the network. The "competition" licences are purchased by subscription 
for  3 days. They are included in the "competition" key which is obtained with Engarde (not
with DiapoEngarde) as other keys.

The competition licences are distributed by the Engarde(PRO MAIN) sessions. These licences also 
allows to run EngardePRO.

When there is no available licence, DiapoEngarde runs in the "Eval" mode. In this mode, 
the size of the display is reduced and the duration of the diaporamas is limited.

                          The "file" menu

#nouv_compe allows the creation of a new competition. 

ENGARDE creates a directory for each competition in which are stored
all the files of the competition. This directory name is displayed
on some windows within ENGARDE.

or in the FFF format.

in order to enter poules and matches, to print reports, etc. on two computers at the same  moment.
In PRO mode, an auxiliary session also allows connections with piste apparatuses.
When modifications are made on fencers (new fencers, modifications of name...), it is necessary
to close and reopen the competition so that Engarde takes them into account.

##aide_sauvegardes

                                     Backups

Backups are automatically made at each important step. It is also possible to select 
to take a backup (#fichier option from the ENGARDE main menu, #deroulement option from 
poules and tableaux menus). This is a good idea when a lot of results have been entered.
Backups are stored in a special directory which has been chosen at the moment of the creation
of the competition. Each backup has a directory with a name including the time of the backup.
To load a backup file, simply load with ENGARDE.

At the end of a competition the backups may be safely deleted from windows 
to recover the space on the disk.

##aide_engarde_xml

                          XML files for FIE

FIE competitions are created from an XML file downloaded from the FIE website. Such a file contains
all the data to run the competition. At the end of the competition, the organiser has to produce
an XML file of the results and to upload it on the FIE website.

                     Creation of a competition from an XML file

Choose "New competition" and "from an XML file". Engarde store fencers, teams and referees in memory.
You have to go to the fencer table and import the fencers (this also imports teams in the case of
team competitions) and to import referees one by one.

                          Loading an XML file in memory

In a competition, it is possible to load in memory data coming from an XML file: fencers, teams or referees.

                     Creation of the XML file of a competition

In the #fichier menu, choose #faire_fic_trans and choose the XML format.
This must be done at the end of an FIE competition and the file must be uploaded on the FIE website.

##aide_engarde_contexte

                               CONTEXTS

Two contexts are available:
- "general contexte" to be used everywhere except France
- "FFE contexte" (Fédération Française d'Escrime) which allows using a file of French clubs
and ligues.

The context defines:
- the name of the national federation,
- the way to define the intermediate affiliation,
- the ages of the different categories (senior, junior, etc.),
- eventually the files of clubs, 'ligues', and countries.

When the first competition is created, if the language is French, Engarde asks the user
to choose the context. If the language is not French, it uses the general context.

When the other competitions are created, Engarde uses the last used context.

The context can be changed with the menu #competition.

##aide_mode

                          MODES

Engarde has 2 possible modes, the BASIC and the PRO modes. In the PRO mode, a session can be
a main session, called "PRO MAIN", the session distributing licences, or in a simple PRO session.
The mode is written in the caption of the window. The colour of the window depends of the mode.

When Engarde is launched on a computer which has a valid key and no session of Engarde is opened,
Engarde runs automatically in mode PRO MAIN, ready to distribute licences if there are licences
to distribute. It is listening to the computers of the network on an UDP port indicated on the
Engarde window. This port is 50100, except if it was not available when Engarde was launched.


The #mode_eng menu allows setting the PRO mode and quitting the PRO mode. When one asks for the
PRO mode and there is no valid key on the computer or a PRO MAIN session is already open, Engarde
opens a window allowing to ask a licence to a computer of the network. At this moment, it is 
necessary to have a PRO MAIN session of Engarde running on the network.
Actions to perform are:
- verify that the UDP port at the top of the window is identical to the reception port
of the PRO MAIN session reception port (generally 50100),
- click on the #ChercheOrdi button to get the list of the computers of the network ready to
provide licences,
- click on a computer proposing licences, this places its IP number in the input area at bottom
- click on #DemanderLice

The #mode_eng mode also allows to activate or deactivate the FIE mode. The FIE mode is obligatory
for FIE competitions, it has the following consequences (from FIE specifications):
- backtrack can be done only after printing a document justifying the backtrack and having it
signed by the President of the Technical Directory,
- main documents have version numbers written under the sub-titles which have to be correctly
managed (see the #document help)
- main documents have an FEI identifier of the form FE_FIE_xxxx (see the #document help)

##aide_parametres

                          PARAMETERS
Data of FIE competitions are provided by XML files which contains value of certain parameters.
For FIE competitions, #domaine_compe #championnat #id fields must not be modified; 
#titre_ligne can be modified.

#titre_reduit is the short name of the competition. It is suggested to use it also for the 
competition directory. It is suggested that a short name with meaning should be used, such as 
London08-SME.

#titre_ligne is the one-line title used by ENGARDE when required.
It is suggested to use a meaningful name, such as:
- London 2000 Senior Men's Epee

#id_reseau is used for the network identification of the competition. Names as 
SME-ind et SME-team for individual and team competitions of Senior Men's Epee

#titre1 is used for the official name of the competition, e.g.: 
"Coupe du Monde - London International Fencing Cup"

#titre2 is used for the weapon, the gender and the category, e.g.: 
"Senior Men's Epee"

#titre3 is used for the date, e.g.: 
"21 February 2008"

#titre4 is used for the location when it is not written in #titre1

##aide_web

                          The Web menu

The Web menu allows automatically sending HTML files to a Website using the FTP protocol
and the FTP.exe application of Windows.

Each time a file is sent, a DOS window is open for running FTP.exe. This window appears
only in the taskbar. It is possible to make it visible by clicking on the taskbar icon.
The DOS window disappears when the file transfer is finished.

When files are automatically sent, it is necessary to observe the DOS windows. In case
of problems (shut down of Internet, overload of the Internet server) many windows are 
opened and no closed. In that case, it is necessary to deactivate FTP, close the windows and
solve the problem before reactivate FTP.

#parametresFTP opens a window allowing to set the FTP parameters
- the path of the ftp.exe application on the local machine must be indicated, it is 
generally c:\windows\system32\ftp.exe
- the path of the directory of the local machine when Engarde save the HTML files must be indicated
- the hostname of the distant machine (web server receiving the files) and the FTP username and 
password.
- the path of the directory of the distant machine where the files will be sent must be indicated 
(it is empty in the case of the root directory).

A button allows activating FTP. It opens a DOS window and runs ftp.exe to get a connection 
and list the distant directory. When the connection is obtained, this window can be close
by typing "Quit". 

It is possible:
- to transfer files with the "Publish xxx" items of the "Web" menu 
- to automatically transfer poules and tableaux by checking a box of the window. In that case,
the frequency of the transfers and the autorefresh period of the files must be chosen. The autorefresh 
period is the time a file will use to call for an update on the client computer (the person viewing
the file on the Internet). Low periods can overload the server.

##aide_cles

                          KEYS

#VoirLesCles opens a window which shows the keys saved on the computer. In the upper part of the window
MAC numbers (physical addresses of the network cards) of the computer are displayed. So is the
reference date which is the date of the day, except if this date has been changed.

Each line of the table corresponds to a key and indicates its start and end date and the MAC 
numbers of the computer on which the key is valid.

The reference date of the computer must be between the start and end dates to have a valid key.
When this is not the case because of an inadequate reference date (due to manual modifications
of the computer date), it is possible to correct the reference date with synchronization. The 
synchronize menu get the date of the day on the Internet.

For having a valid key, at least one of the MAC numbers of the key must be one of the active MAC 
number of the computer.

The line of a key also indicates the state of the key, the type of the key
and the number of licences. 

#InstallerCle allows installing a key which has been purchased. The window contains fields for
the client identifier and the purchase code (this information is provided at the purchase time).

#AcheterCle leads to the Engarde website in order to purchase a key.

##aide_tables
                       TABLES


Tables are made up of entities (fencers, referees, clubs, etc) or
information about a competition (parameters, formula, etc.).

They contain all the information on these entities. They do not allow the user
to enter or modify the entities directly, but the entity can be selected from
the table, and the menu or short cut used to edit or create new entities.

A "DOUBLE CLICK" on a line means "edit" the entity on that line.

It is possible to modify the column widths of the tables and define the 
order of the columns:
- to change the width of a column, place the mouse cursor at the right of 
the cell and move the right hand to the end of the cell
- to change the order of the columns, drag the column title (from the centre 
of the title) to the required position.

The column order in a table is the one used for data entry.
When a bar-code reader is used the fields must be placed in the order that 
the fields are being read, for example name, first name, licence.

During the entry and modification of entities contained in the tables,
the information is not continuously saved. The File #enregistrer option 
allows the data to be saved immediately.

It is possible to select several entities (several lines) with the mouse to 
apply an action such as #supprimer or #present or #absent.


Actions on tables are:

* #fichier menu
- #exporter_table to export table data in a text file
- #importer_table to import entities in the #reserve

Depending on the table entities, #fichier menu can include other items dealing with reading/writing files.

* #reserve menu

The #reserve menu allows users to load into memory files containing details
of fencers whose details are to be loaded into a competition.

- #charger to load a file into memory (several files can be loaded into memory)
- #vider to empty the memory
- #effectif to display the number of fencers held in memory
- #importer_un to import a fencer from memory into the competition
- #importer_certains to import a range of fencers between two supplied
values into the competition.
- #importer_tous copy all the fencers from memory into the competition


* #document button
- fencers: For FIE competitions, print this document after moving to version 1 (or more when 
necessary), this leads to the official document FE_FIE_0001
- referees: For FIE competitions, print this document after moving to version 1 (or more when 
necessary), this leads to the official document FE_FIE_0002

* #edition menu
- #saisir to open a window to create new entities
- #modifier to modify the entity on the selected line
- #supprimer to delete the entities on the selected lines. Entities that
are referenced by other entities may not be deleted.
- #chercher to search for an entity in the table by entering the name.
- #agglomerer allows to create one entity from two. For example, if there
 are two clubs that were, in fact one club these can be merged as one club.
- #colonne1fixe allows to toggle the fixing of column 1. This allows column 1 to be viewed 
when horizontal scrolling.



* #ordre buttons
The #ordre buttons enables sorting by alphabetic order, by structured order or 
by fencer number.

* #ordre menu

In addition to the sorting buttons, other options are available with the menu to select the order of the entities.

The table can now be sorted by internal number. When entered, the last entered
records are placed at the end.

* #presence buttons
The #presence buttons enables marking an entity as present or absent with a single click.

* #presence menu
The #presence menu adds options to mark presence of the entities.
- #present to mark the selected entities as present
- #absent to mark the selected entities as absent
- #presentpardefaut to mark, at the start of data entry, all the entities as present
- #absentpardefaut to mark, at the start of data entry, all the entities as absent

It is possible to export fencers, teams and referees in text files. It is also possible
to import files in this format.
These options are found in the #fichier menu of each table. The files are made up of lines
made up of fields separated with";". The first line contains the names of the fields.

When importing a text file the contents are loaded in memory.
To use this data the #reserve menu should be used.

In the team table it is possible to calculate the rankings (or points) of the teams
starting from those of the fencers. (Engarde takes the best three).
This is done from the Team menu.

In the fencers table, when running a team competition, it is possible to 
automatically create teams with just a team name.

* The EngardeSmart menu is used to send data from the table to the EngardeSmart service.

##aide_recherche

                     SEARCH

Searching is incremental. By typing slowly the letters of the field being
searched upon at each step the list of entities that match the typed letters
are displayed (as long as there are no more than 1000 entities of that 
type).

In the mode #commencant_par the entities whose names begin with the typed
text are displayed.
In the mode #contenant the entities whose names contain the typed text in 
any part of the name is displayed.
In the mode #exacte the search returns only exact matches, including Upper 
and Lower case letters.
In the mode #approchee a loose match is performed.

A record is selected by 'clicking' once and selecting OK or by 'double
clicking' on a record.

##aide_saisie_gene

               DATA ENTRY AND MODIFICATION SCREEN

This screen allows entry and / or modification of fencers, clubs, countries,
i.e. all data other than the competition details (parameters, formula, 
etc.).

Movement from one field to the next is performed using the 'Tab' key and to 
the previous field using the 'shift' key together with the 'Tab' key.
A specific field can also be selected selecting it with a single click using 
the mouse.

A record or field can be erased from the "Field" menu (or hitting "shift"+"Del").

A record is exited using #fermer. If a record has been changed then this record
can be exited either by closing the record or moving to another record using 
an option from the #fiche menu.

When working with tables:
- when a record is being entered and the user chooses to exit when there is no
name then the record is not saved.
- when a record is being modified then an exit is not permitted when there is no
name.

Certain fields have a default value that is set by ENGARDE at the start of the
data entry (for example: present).

When a field has a default value and the user deletes the value of the field
ENGARDE resets the field to the default value.

Data Entry:
- In the name, first name, etc. it is not necessary to use upper case letters
as the text is placed in upper case as required by ENGARDE
- In the integer numeric fields only digits are accepted; if the keyboard
settings are correct the program should also accept the text keys enabled using
"num lock".
- In the decimal numeric fields the decimal point is "." If a comma "," is typed
this is automatically translated in a ".".
- In fields with prompts (e.g. presence (present/absent) the field is set by
hitting the first letter of the required value.
- affiliations are not directly entered in the field. These are found using a
special window, where they can be selected from lists, or new values 
entered.

##aide_saisie_divers

                   DATES

In date fields the separator between day, month and year is "/"

The characters ": , ; ." may also be used, but are all translated into "/"

If the year is <20, ENGARDE adds 2000 : for example, 3 becomes 2003

IF the year is 20<= and <100, ENGARDE adds 1900 : for example, 68 becomes 1968

Dates can be incomplete - just the year or month and year.

Dates can also be entered in the formats: ddmmyy and ddmmyyyy.


                   STATUS

The status of a fencer is used to indicate the passing of a particular event
(abandonment, exclusion or scratch) that occurs during a competition. Care must
be taken if this happens during the competition. If this occurs, then the status
(abandonment, exclusion or scratch) must be indicated when the results of a 
phase are being entered.

Giving the 'abandoned' status to a fencer who has qualified for the next phase
leaves the fencer in the overall ranking.
Giving the 'scratched' status to a fencer who has qualified for the next phase,
but who does not check in for that phase, will remove the fencer from the
overall ranking.
Giving the 'excluded' status to a fencer who has qualified for the next phase
removes the fencer from the overall ranking and replaces the fencer
with a blank space.


                   AFFILIATIONS

ENGARDE supports three affiliations for fencers: club, "ligue", and country.
"ligue" is the intermediate affiliation between club and country. This is a
structure used in France, but can also be used in other countries, for 
example "state".

It is not required to use all three levels of affiliation. You can use only
country, or club and country without ligue, etc.
Affiliations are used to perform separation when creating poules and tableau 
(if allowed in the chosen formula). It is only necessary to enter those fields 
used by the chosen formula.

Affiliations are not directly entered in the field. These are found using a
special window, where they can be selected from lists, or new values 
entered.

If an affiliation already exists, then you must not re-enter it.

You must supply the first affiliation. If possible, the other affiliations are
derived from the first one. If no subsequent affiliations exist, then these can
be created. For example, when entering a club for a fencer the "ligue" and
country are automatically associated with the fencer as they had previously 
been defined.

When an affiliation is changed both the 'parents' are updated as appropriate,
and any 'children' are blanked. For example, if the "ligue" of a fencer is
changed, then the country (parent) is updated with the appropriate country 
and the club (child) is blanked.

The #affiliation options are:
- #chercher_compe to search within a competition
- #chercher_fic to search in memory (you must have first loaded a file into
memory))
- #saisir to enter affiliations in the affiliation window
- #effacer_champ to clear the affiliation field

##aide_cre_poules
                            CREATING POULES

Poules are created (button #const_poules or control menu from the "#poules" window) at the start of the competition using the
fencers who are present, or after a round using the fencers who qualified
from the round and exempt fencers entering at that point.

The poules are created following the formula that had most recently been
selected. The formula can be modified at the point of creating the poules
(ENGARDE presents the main parameters and allows them to be modified).

When creating poules the exempt fencers are set to one side (there can be)
a barrage) then the other fencers are placed in the poules using the separation
that has been selected. ENGARDE tries to avoid conflicts of all the
criteria indicated in the formula. When it is not possible, then the priority
is given to the first criteria indicated in the formula and then the second.

ENGARDE selects at random when the rank of fencers is exactly equal.
After having created the poules, when there is separation, ENGARDE presents the
distribution of fencers within the poules.
The poules are displayed in the "#poules" window and ares also available in the report menu.
Additional details (#heure, #piste, #arbitre) will be automatically updated in the window.

When the distribution is not satisfactory the poules can be redone to obtain
a better distribution. This can produce different results when there are 
exactly equal fencers in the ranking.

When it has been selected in the formula, if progressive entry is to be used,
ENGARDE requests that the fencers are selected for the first round of poules:
fencers between fencer number values (or number of points). It then requests
that fencers who enter in subsequent rounds are selected in the same manner.

##aide_clas_poules
                       RANKING OF THE POULES

The ranking can be performed (item #classement of menu #deroulement) at the end of 
each round.
It is performed following the selected formula. In the "classic" formula
the fencers are selected based on their indicator, and the best "X" are 
selected for the next phase. There can be a barrage.

In the last round the selection for the next phase can be done using the 
results of the two previous rounds.

It is possible to do a special ranking (item #clas_special of menu #deroulement),
in that case, the main criterion for classifying fencers is the rank in the poule.
If Martin is first in a poule, with 3 victories for 5 matches, and if Lefevre is second
in a poule with 4 victories for 5 matches, then Martin is before Lefevre. 

After selection, ENGARDE creates the ranking for the next round. To do this,
it takes the exempt fencers (if there are any) and the qualifying fencers.
The ranking for the next round can be on one or two rounds (it must be on
two rounds if the selection for the round was based on the results of two
rounds).

##aide_formule
                        FORMULA

The formulae available in ENGARDE are:

- poules throughout the competition
- poules followed by tableaux without repechage
- tableaux from the start of the competition without repechage

The poules are organised in rounds of poules, with the number of poules and
fencers per poule based on the input parameters of the competition. Each round
can have exempt fencers. The number of exempt fencers can change from one round
to the next. There are no real constraints on the size of the poules - ENGARDE
can deal equally well with poules of 3 as poules of 20 fencers.

A tableaux, or direct elimination, is organised as series of tableaux, with 
the number of phases based on the input parameters of the competition. Each 
stage of a tableau can contain exempt fencers. The number of exempt fencers can 
change from one stage of a direct elimination to the next.

When entering the formula of a competition for the first time, ENGARDE 
proposes the following formulae for individual competitions:
- #tab_dir_sans3
- #tab_dir_avec3
- #tableau_inacheve : tableau with possible exempt fencers and stopped before the end
- #formu_fie_seniors with a preliminary tableau with 32 exempt fencers and a main tableau of 64
- #formu_fie_juniors : identical to #tab_dir_sans3
- #autre_formule allows other possibilities (load the template or create a new formula).

A formula can always be modified for the part of the competition that has
not yet taken started.

The formula window contains:
- a #document menu prints the formula. For FIE competitions, print this document after moving
to version 1 (or more when necessary), this leads to the official document FE_FIE_0004
- four panels described below. The details of the panels can be displayed or hidden with a button.

The 'Main Options' panel provide an access to the base
information of the competition.
The information is:
- number of fencers,
- original ranking,
- number of rounds of poules,
- number of hits on the matches in the poules,
- number of rounds for qualification (1 or 2),
- number of rounds for ranking (1 or 2),
- number of phases in the direct elimination,
- number of hits in the matches of direct elimination.

The "Formula for the poules" panel allows the user to enter information on
the rounds of poules.
The panel contains a table, with one line per round.
By 'double clicking' one can change the information of a round. When the data
is inconsistent ENGARDE changes the data.

The "Formula for the direct elimination" panel allows the user to enter information
on the phases of tableaux.
On selecting this button a table is displayed, with one line per phase of the
tableaux.
By 'double clicking' one can change the information of a phase of the tableau.
When the data is inconsistent ENGARDE changes the data.
ENGARDE supplies the number of fencers at the start, and calculates the number
of fencers at the end.

The information of a round of poules, or a phase of tableaux are presented to
be modified (if required) at the moment when the round of poules (or phase 
of tableau) are ready to be created.
It is not necessary for the poules or tableau to be complete.

The 'report' button provides the option to produce a report that details the
formula in a format that can be printed.

The 'Load' button enables loading a formula among the list of available formulae. For each formula a description is given.

The 'Important change' button allows the creation or modification of the formulae of 
tableaux.
To understand how this option functions, it is advised to load the supplied 
formulae and study them.
This option must be used with care. All new formulae must be completely 
tested using the simulation options prior to using them in a competition.

The basis of the formula is as follows:

- initial ranking: perform the ranking based on the initial ranking.
- initial ranking in groups: rank all marked 1 before those marked 2, etc.
and at each stage rank the fencers with the initial ranking
- natural progression: place each fencer in the position that they have 
achieved.

It is not normally possible to modify the destination of the tableaux, they 
are calculated automatically by ENGARDE.

It is not normally possible to modify the groups of rankings of the 
tableaux, they are calculated automatically by ENGARDE.

To save a formula as a template, it must be saved using the #enregister_modele button. The file has to be saved in the "mesFormules" folder which is proposed and which contains the custom formulae. When man load a formula, Engarde also proposes the custom formulae.

##aide_abandon

ABANDONMENT, FORFEIT and EXCLUSION

The windows for entering poules or matches are used to indicate when a fencer stops a poule or a match before the end. The processing carried out by Engarde follows the FIE rules.

Three reasons are distinguished by the FIE for these stops:
- EXCLUSION: it is a stop for disciplinary reason (disciplinary black card).
- ABANDONMENT: it is a medical stop.
- FORFEIT: it is a stop for another reason, including a black card for non-presentation on the piste or for defective material.
The reason for the distinction between abandonment and forfeit is the fact that the FIE wants to make stops statistics for medical reasons. Otherwise, their treatment is identical.

The treatments for the ranking of the current round and the final ranking are:
- EXCLUSION: the fencer does not appear in these rankings.
- ABANDONMENT or FORFEIT: the fencer appears in these rankings. If he/she fenced at least one whole poule or match of the tableau, he/she is classified as if eliminated at that time. In other cases, he/she is placed at the end of the ranking with no rank.

When the pistes are connected, the information can be communicated with the remote control of the piste apparatus, if it includes the Exclusion, Forfeit and Abandonment functions.

It is possible to change the status of a fencer in the fencer table. This is to be done in the following cases:
- a fencer is qualified for the next round (or a tableau to be created) and is in Exclusion, Abandonment or Forfeit. By giving him/her this status Exclusion, Abandonment or Forfeit in the table of the fencers, he/she will not be integrated in the next round (or a tableau to be created).
- following an error, a fencer has an Exclusion, Abandon or Forfeit status. It is possible to give him/her the Normal status. You may have to modify some results, or redo the poules, or redo the tableau.

The menu "Document | Statistics | Statistics (abandonment...)" indicates the Exclusions, Abandonment or Forfeit and their treatments.

In the XML files, the status for « Exclusion », « Abandonment » or « Forfeit » are respectively E, A and F.
